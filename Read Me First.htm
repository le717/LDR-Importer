System Requirements

  • Blender 2.63 or higher
  • ldraw_import.py (included with this download)
  • LDraw System of Tools
  • MLCad, LeoCAD, or Bricksmith (all optional)

Setting the LDraw Folder Path

  • On Windows, open ldraw_import.py in Notepad or Notepad++. On OS X, open the script in TextEdit (be sure to use plain text mode!), or TextWrangler.

  • Look for line 46, which will say
    LDrawDir = "C:\LDraw"
    Replace the text with the path to your LDraw System of Tools.
  • Browse to the Blender 2.6 Python startup folder.

    • Windows

  • Depending on your version of Windows and Blender, the folder is located at 
    %ProgramFiles%\Blender version number\scripts\startup
    OR 
    %ProgramFiles(x86)%\Blender version number\scripts\startup

    • Mac OS X

  • Open Finder, click Go > Go to Folder..., and type 
    ~/Library/Application Support/Blender/Blender version number
  • Click File > Create Folder, and create a folder called scripts.
  • Create another folder called startup in the scripts folder.
  • Your folder path should look like 
    ~/Library/Application Support/Blender/Blender version number/scripts/startup
  • Copy ldraw_import.py to this folder.

    That's it! You have successfully installed the Blender 2.6 LDraw Importer script.

    • Known Bugs


    1. If a brick cannot be found, entire import stops.
    2. Bricksmith LDR files do not properly import, due to different layout than normal LDR files.
    3. Blender 2.65 crashes if user imports multiple models and available RAM is too low to continue. Can never be fixed, due to various computers with different amounds of RAM.
    4. Alpha colors are not supported.
    5. Find a way to set the LDraw Brick Library path in Blender itself.
    6. Rare bug where Blender 2.63 will not import any brick colors after already importing a model; may have already been fixed.
    7. Rare bug where Blender 2.63 will crash upon importing a model; may have already been fixed.
